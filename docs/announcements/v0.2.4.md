## Reddit

**Title:** GopherTrunk v0.2.4 — APRS + POCSAG end-to-end, Radio IDs panel, P25 on the wideband channelizer (CC + voice on one SDR)

**Body:**

**GopherTrunk v0.2.4 is out.** GopherTrunk is a pure-Go digital-trunking
radio scanner for RTL-SDR / HackRF / Airspy / Airspy HF+ — P25 (Phase 1 +
2), DMR (with AMBE+2 voice), NXDN, Motorola Type II, EDACS, LTR, MPT 1327,
dPMR, D-STAR, YSF. No CGO, no `librtlsdr` / `libhackrf` / `libairspy` /
`libusb` at build or runtime, single ~10 MB static binary for Linux, macOS,
and Windows, with a headless daemon, a Bubbletea TUI cockpit, and a
browser-based web operator console.

This is the **Phase-3 / Phase-5 / Radio-IDs feature-density** follow-up to
v0.2.3 (which itself shipped the iqtap broker + Constellation /
CC Activity / Bookmarks / spectrum click-to-tune / rigctld / remote
`rtl_tcp` panels in one batch). The wideband channelizer also grew P25
support and a virtual voice pool so one SDR can cover P25 CC + voice
end-to-end.

What's new since v0.2.3:

- **APRS end-to-end protocol layer + scaffold (#384, #390).** New
  `events.KindAPRSPacket` bus event, `aprs_log` SQLite table,
  `GET /api/v1/aprs/packets?limit=N` REST endpoint, and `/aprs` web
  panel rendering the live packet list (received time, src → dst +
  path, type, body, lat/lon, CRC-OK flag). Plus a pure-Go AX.25 frame
  parser and APRS info-field decoder covering positions / messages
  (with ack/rej + bulletins) / status / object / weather / telemetry /
  Mic-E. The Bell-202 AFSK DSP receiver (FM demod → HDLC de-stuff →
  frame delivery) is the final piece, landing as a focused follow-up
  PR. See [docs/aprs.md](https://gophertrunk.org/aprs.html).
- **POCSAG paging closes end-to-end (#378).** v0.2.3 shipped the
  BCH(31,21) FEC + codeword wrapper + numeric / alphanumeric decoders
  (#372) and the syncer + page assembler + bus / log / REST / web
  panel scaffold (#373). v0.2.4 wires the DSP receiver (FM demod →
  rational resampler → integrator-and-slicer → bit syncer) so a tuned
  SDR's IQ now flows end-to-end into the pager bus event without any
  remaining plumbing. A new `paging.pocsag` YAML section pins SDRs to
  paging frequencies; the daemon retunes on startup, subscribes to
  the iqtap broker, and runs one receiver per configured entry as a
  non-essential spawn so a misconfigured paging frequency doesn't
  bring down the trunking pipeline. See
  [docs/pocsag.md](https://gophertrunk.org/pocsag.html).
- **Radio IDs as first-class entities + standard P25 talker-alias
  voice-channel decoder (#387, #389, closes #376).** New `RIDDB`
  operator-configured alias catalogue mirroring `TalkgroupDB`:
  per-system `rid_alias_file` (CSV or JSON) carrying `Decimal/DEC/ID`
  plus optional `Alias`, `Description`, `Tag`, `Group`, `Owner`,
  `Priority`, `Lockout`, `Watch`, `Icon` columns. New REST routes
  (`GET /api/v1/rids`, `GET /api/v1/rids/{id}`, `GET
  /api/v1/rids/{id}/history`, `PATCH /api/v1/rids/{id}`), new gRPC
  `RIDService`, new `/rids` web panel with configured ∪ live merge,
  last-50-calls detail modal, and write-mode mutation controls. CC
  Activity RID chips are clickable links into the detail view. Plus
  the standard TIA-102.AABF P25 voice-channel talker-alias LC decoder
  (LDU1 LCOs 0x15 HEADER / 0x16 BLOCK1 / 0x17 BLOCK2 reassembled
  into a `KindTalkerAlias` event), closing the second half of #376.
  The existing Motorola vendor TSBK form is unchanged. See
  [docs/radio-ids.md](https://gophertrunk.org/radio-ids.html).
- **Wideband channelizer hosts P25 Phase 1 + Phase 2 CCs alongside
  DMR T2/T3 (#385).** A single SDR pinned to a centre frequency can
  now host a P25 trunked control channel inside the wideband
  channelizer. The per-channel wiring uses a small protocol-agnostic
  `narrowbandReceiver` interface; P25 Phase 1 honours
  `p25_phase1_demod_mode` (C4FM vs CQPSK / LSM) and any
  operator-supplied `P25BandPlan`, and P25 Phase 2 reuses the
  existing trellis / RS / interleave / scrambler / clock-mode knobs
  and the PN44 seed derivation so a wideband CC tap decodes
  identically to a dedicated CC dongle.
- **Virtual voice pool on the wideband dongle (#386).** A wideband
  dongle can also follow trunked voice grants whose frequency lands
  inside its IQ window — DMR Tier III, P25 Phase 1, P25 Phase 2 —
  without a separate `role: voice` SDR. New
  `internal/sdr/wbvoice.VirtualTuner` subscribes to the wideband
  dongle's iqtap broker on demand and runs a single-tap DDC at the
  (target − wideband) offset, emitting 48 kHz IQ into the composer's
  existing P25 / DMR voice chains. **One SDR end-to-end for any
  trunked system whose carriers fit in a single 2.4 MHz band.**
- **Wideband spectrum view (#377).** Wideband-only deployments
  (single SDR, `role: wideband`, multiple T2 systems) couldn't
  render the spectrum waterfall because the engine consumed
  StreamIQ from the raw device. The wideband engine now routes
  through the iqtap broker, mirroring the CC decoder wiring, so the
  spectrum panel works on wideband-only deployments. Also fixes
  `sample_rate_hz=0` on emitted frames.
- **Windows RTL-SDR cold-boot stalls now self-recover (#382, #393,
  issue #333).** Clone dongles and power-marginal hubs latch the
  first vendor-OUT write, then NAK the byte-identical second write
  with `ERROR_GEN_FAILURE (0x1F)`. The Linux equivalent (`EPIPE`)
  was already covered; the Windows path wasn't. WinUSB
  `Transport.Reset()` now matches `libusb_reset_device` (clear-halt
  the control endpoint, drop the WinUSB handles, re-open via
  `CreateFile` + `WinUsb_Initialize`), the open-time bring-up
  envelope allows up to two such resets with 100 ms / 200 ms backoff
  for clones that need two settles, and `ErrPipeStalled` keys the
  existing retry envelope. Healthy dongles still open with zero
  resets and zero delay.
- **r82xx PLL widened to 268 nint encoding limit (#391, closes
  #264).** The overflow guard used `0x3F + 13 = 76`, which ignored
  that si's 2 extra bits also encode part of nint (real cap is
  `13 + 4*0x3F + 0x3 = 268`). With R820T / R820T2's 28.8 MHz xtal
  the VCO range capped nint near 67 so the bug was latent; PR #266's
  correct R828D xtal (16 MHz) halves `pllRef` and pushed nint up to
  ~121, so the guard rejected V4 dongle tunes above ~140 MHz. Fixed.
- **CC Activity super-group + member-count rendering (#392, closes
  #374) + actionable "voice pool empty" diagnostic (#383, closes
  #379).** `eventToDTO` missing `trunking.Patch` case meant patches
  rendered "super-group 0 · add" with no member counts; new
  `PatchDTO` fixes the wire shape. And when an operator booted with
  a trunked system but no `role: voice` SDR, every grant logged
  "voice pool full but no actives" — misleading because the pool
  was empty, not full. An actionable one-shot startup WARN now
  surfaces the problem before the first grant arrives, pointing at
  `docs/hardware.md`.
